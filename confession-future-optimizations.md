# Confession 項目後續優化備忘

本文件記錄從論文正文移出的項目後續優化方向。它用於後續產品與研究迭代備忘，不作為 `confession-thesis.md` 的正文內容。

## 評估資料與實驗設計

- 擴充真實項目樣本與人工標註集，覆蓋更多框架、語言特性與專案規模。
- 建立獨立標註流程，記錄標註者、判定依據、衝突處理方式與可重跑資料版本。
- 將 AST 候選層、LLM 審查層與人工確認結果分開統計，避免不同層級指標混在一起。

## Agentic 成功路徑觀測

- 在固定模型、有效憑據、固定配額與固定快取初始狀態下重跑 agentic 成功路徑實驗。
- 分別記錄 Planner、Skill/MCP、Analyst、Critic、Judge 各階段耗時、token 成本與失敗原因。
- 將 baseline、agentic 成功、agentic 回退三種路徑拆開觀測，便於比較成本與收益。

## 跨文件語義分析

- 在現有 AST 候選基礎上加入調用圖、資料流與代碼切片能力。
- 擴充 SQL 注入、XSS、權限繞過等跨函式與跨檔案場景。
- 建立 framework-aware 規則，例如 ORM 查詢構造、模板渲染、框架特定逃逸與 sanitizer 判定。

## LLM 輸出一致性

- 強化模型輸出的證據引用，要求每個判斷對應具體檔案、行號與程式碼片段。
- 加入一致性檢查與結構化驗證，降低格式異常、欄位缺失與信心分數不一致問題。
- 對高風險結果加入二次審查策略，例如 Critic/Judge 交叉確認或規則回查。

## 協作與遠端彙總

- 探索在 FileStore 本地模式之外提供團隊協作與遠端彙總能力。
- 設計多使用者審查狀態同步、審查責任分配與審查歷史查詢。
- 評估 SARIF、Dashboard 與 CLI 報告在團隊安全治理流程中的整合方式。
